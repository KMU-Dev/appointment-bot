## Kubernetes 是什麼？

Kubernetes 是一個可移植的、可擴展的開源平台，用於管理容器化的工作負載和服務，藉由配置文件來達成容器自動化。Kubernetes 擁有一個龐大且快速增長的生態系統。Kubernetes 的服務、支持和工具具有極高的可用性，目前在各大雲端平台為核心管理架構，被譽為「網路界的Linux」。

## 為什麼需要Kubernetes，它能做什麼?

容器是打包和運行應用程序的好方法。在生產環境中，你需要管理應用程序的容器，並確保不會因為意外而停機。舉例來說，如果一個容器發生故障，則需要啟動另一個容器。如果讓處 Kubernetes 來處理這件事，管理應用程序是不是變得更容易了？

這就是Kubernetes 來解決這些問題的方法！Kubernetes 為你提供了一個可彈性運行分的佈式系統框架。Kubernetes 會滿足你的擴充要求、故障轉移、不同的部署模式等。

Kubernetes 為你提供：

* **服務發現和負載均衡**

  Kubernetes 可以使用 DNS 名稱或自己的 IP 地址公開容器，如果進入容器的流量很大， Kubernetes 可以分配網絡流量，從而使部署穩定。

* **存儲系統**

  Kubernetes 允許你自動掛載你選擇的存儲系統，例如本地存儲、公共雲提供商等。

* **自動部署和滾動升級**

  你可以使用 Kubernetes 描述已器的所需狀態，它可以用您定義的速率將實際狀態更改為您期望的狀態。例如，你可以利用Kubernetes 來為你的部署創建新容器， 刪除現有容器並將它們的所有資源用於新容器。

* **自動完成資源計算**

  Kubernetes 允許你指定每個容器所需 CPU 和內存（RAM）。當容器指定了資源請求時，Kubernetes 可以做出更好的決策來管理容器的資源。

* **自我修復**

  Kubernetes 重新啟動失敗的容器、替換容器、中指無法正確回應的容器（條件可由使用者自訂），並且在準備好服務之前不會將它公開給客戶端。

* **密鑰與配置管理**

  Kubernetes 允許你存儲和管理敏感信息，例如密碼、OAuth 存取金鑰和 ssh 密鑰。你可以在不重建容器鏡像的情況下部署、更新密鑰和應用程序配置，也無需在記憶體中暴露密鑰，增加資安風險。
